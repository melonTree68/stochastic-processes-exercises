# 《计算机科学中的随机过程》开源习题库安全阅览守则

欢迎来到本习题库。本仓库是《计算机科学中的随机过程》一书的配套习题。为了确保您的马尔可夫链能够在有生之年收敛，并在提交 PR 后您的发际线不会产生不可逆的退化，请务必严格遵守以下规则。

**绝对原则**：如果你在访问本网站时发现屏幕上的字迹开始变得模糊且眼睛感到酸涩，请立刻闭上双眼，放缓呼吸，保持两分钟后，你会变得安全且舒适。

## 第一部分：关于题目本身
每一章习题分为两部分：练习和问题。

练习部分的题目相对基础，作者保证它们都是由人类构思，且存在凡人可解的答案。您可以放心作答。

问题部分更强调综合能力。在作答这部分时，某个状态空间的大小达到了 $2^n$ 且毫无对称性可言，请立刻放弃寻找精确解，并默念三遍“我相信选择公理”。千万不要试图求出那个不存在的稳态分布，除非这是解题的唯一思路。

仓库中没有第零章。如果您在目录中看到了 chapter_00，或是看到了任何不合理的现象，请立即通过邮件提醒管理员。

## 第二部分：Fork 与同步
为了贡献你的解答，你需要将仓库克隆到本地。请严格按照以下步骤操作，切勿错漏。

仓库唯一安全地址：https://github.com/yuchen-he00/stochastic-process-exercises。若发现 URL 中的 yuchen-he00 变成了其他无规律字符，请立即停止行动，并重新阅读本页面规则。

克隆你自己的仓库：git clone <你的仓库地址>。

把原仓库设为 upstream：
git remote add upstream https://github.com/yuchen-he00/stochastic-process-exercises.git

开始做题前，务必同步到最新主分支：
git fetch upstream
git checkout main
git rebase upstream/main
警告： 如果在执行 git fetch 时，终端在10秒钟内没有反应，请不要着急，这是无害的，请不要寻找管理员，必要时可以寻求AI agent的帮助，但要小心，过度依赖AI agent可能会导致幻觉的产生和能力的退化，这种影响是可逆的，但请尽量不要出现这种情况，会很棘手。

## 第三部分：寻找对应题目与解答文件
请在 docs/exercises/ 里寻找题目。

每道题下方都有一行 --8<--。这是“剪刀”的标记。它标记了上一位贡献者理智断绝的地方，顺着它，你可以找到对应的解答文件。

例如：--8<-- "solutions/chapter_01/exercises/exercise_coupon_collector.md"

绝对不要去寻找没有 --8<-- 标记的解答文件。不要好奇那些没有被剪刀标记的题目背后藏着什么。

## 第四部分：追加你的内容
把下面代码块直接复制到对应文件末尾，并在对应位置写下你的解答。

```markdown
??? success "来自[Your Name]的解答"

    你的解答写在这里。

??? note "来自[Your Name]的想法"

    你的补充思路或者评论写在这里。
```
严禁修改文件内已有的任何内容。 原本的解答是由不同的贡献者写下的，篡改会被视为极大的挑衅。

请严格使用 success 或 note 标签。永远不要在代码块中使用其他奇怪的标签，我们不为召唤出的后果负责。

## 第五部分：提交 Pull Request（接受审判）
提交修改：git commit -m "add solution for xx, by xx"。请保持 Commit Message 的简短与客观，不要在里面添加任何无关信息，会干扰管理员的理智，导致系统崩溃。

推送到你的仓库并提交 PR。确认 base branch 为 main。其他分支是不可靠的，请不要信任。

作者会在 1～3 天内审核你的 PR。

如果你的 PR 在点击提交后的一分钟内被通过，请不要相信，更不要赞美管理员的效率，记住，这不是管理员的操作，管理员永远不可能如此高效。

在认真答题和思考的过程中，管理员会顺着网线过去保护你，请不要担心任何安全问题。
